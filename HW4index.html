Progressive Web Applications (PWA)

Definition

Progressive Web Apps (PWAs) are websites built with modern web technologies that offer an app-like experience, including offline capabilities, fast performance, and push notifications. They are installed directly from the browser to the home screen, bypassing app stores.

History

When Steve Jobs first floated the idea of a “web app” in 2007, he was ahead of the times. The demand for native apps and a developer’s kit was too strong to ignore, and so native apps acquired from an App Store became the standard for delivering applications to mobile devices. But less than a decade later, in 2015, Google coined the term “Progressive Web Apps” to describe an updated version of Jobs’ web app concept for reliable, fast applications that can run on varied devices, work offline, and are developed using a single codebase. The development of Service Workers and Web App Manifests around the same time made PWAs possible by providing offline functionality and installability.

Key Characteristics

Reliable, Responsive, Flexible - PWAs must always load, even when offline, to provide a consistent user experience. They run on any browser, and varied devices (desktop, tablet, mobile). This is accomplished using a JavaScript Service Worker running in the background, to store key resources and enable push notifications. Because they are web-based, PWAs are discoverable by search engines, and updates are pushed through the web server.

Engaging User Experience - The web app manifest specifies to the browser how the app appears and is launched (home screen icons, theme colors, page to load when the app is launched, the screen orientation, etc.). This allows the PWA to have a native app-like feel. PWAs can be installed by users onto their home page, without going through an app store.

Development Considerations - Progressive web apps must be served via HTTPS to ensure user privacy, security, and content authenticity. A significant benefit of PWAs for developers is that they work across multiple platforms using a single codebase.
